# タスク一覧画面_総合テスト計画書

## 1. 目的
タスク一覧画面の改修内容について、画面表示、操作、関連画面への影響を確認し、
想定どおりに動作することを確認する。

## 2. 対象画面
- タスク一覧画面

## 3. 関連画面
- 案件一覧画面
- タスク詳細画面
- 新規タスク作成画面
- CSV出力ダイアログ

## 4. 改修概要
今回の改修では、主に以下を確認対象とする。
- 画面タイトルの見直し
- 説明文の追加または調整
- 戻る導線の追加または調整
- タスク一覧表示部のUI調整
- 状態表示の調整
- CSV出力関連UIの確認
- 共通CSS/JS影響確認

## 5. 対象範囲
- 画面初期表示
- 文言表示
- レイアウト表示
- 一覧表示
- 状態表示
- ボタン操作
- 画面遷移
- CSV出力
- エラー表示
- 共通部品影響

## 6. 対象外
- 性能試験
- 負荷試験
- セキュリティ試験
- ブラウザ横断の詳細試験
- API内部ロジック単体確認

## 7. 実施環境
- OS: Windows
- ブラウザ: Edge / Chrome
- 実施環境: 開発環境
- データ: テスト用案件・タスクデータ

## 8. 実施条件
- ログイン済みであること
- 対象案件が存在すること
- タスク0件 / 1件 / 複数件のデータを準備していること
- 各状態（ToDo / Doing / Blocked / Done）のデータを準備していること
- CSV出力対象データが存在すること

## 9. 確認観点
- 文言
- レイアウト
- 一覧表示
- 状態表示
- 操作
- 画面遷移
- エラー表示
- CSV出力
- 共通CSS/JS影響
- 関連画面影響

## 10. 判定基準
- 期待結果どおりであること
- 表示崩れがないこと
- 業務上誤解を招く表示がないこと
- 関連画面へ影響が出ていないこと

## 11. 成果物
- 確認観点一覧
- 総合テストケース表
- 実施結果記録
- 必要に応じて不具合メモ
