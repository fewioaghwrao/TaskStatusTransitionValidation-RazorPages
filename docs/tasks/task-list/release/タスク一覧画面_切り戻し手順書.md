## 2. 目的

本書は、タスク一覧画面の本番反映後に重大な不具合が発生した場合に、速やかに反映前の状態へ戻すための手順を定めるものである。

※ 本資料は学習・模擬案件用に作成したものであり、環境情報の一部は仮置きである。

---

## 3. 切り戻し対象

原則として、今回反映した資材一式を切り戻し対象とする。  
影響範囲が明確であり、整合性が保てる場合に限り、対象資材のみ個別切り戻しを可とする。

### 対象資材
- `TaskStatusTransitionValidation.RazorMock/Pages/Projects/Tasks/Index.cshtml`
- `TaskStatusTransitionValidation.RazorMock/Pages/Projects/Tasks/Index.cshtml.cs`

- `TaskStatusTransitionValidation.RazorMock/wwwroot/css/tasks.css`
- `TaskStatusTransitionValidation.RazorMock/wwwroot/js/projects-tasks.js`

※ `tasks.css`、`projects-tasks.js` は今回反映対象に含まれる場合のみ切り戻し対象とする。

---

## 4. 切り戻し判断条件

以下のいずれかに該当する場合、切り戻しを検討する。

- タスク一覧画面が正常に表示されない
- 案件名を含むタイトル、説明文、戻るリンクが正常に表示されない
- タスク一覧が正常に取得・表示されない
- 0件 / 1件 / 複数件 / 長文表示で重大な崩れがある
- 状態表示が不正である
- タスク詳細画面または新規タスク作成画面への遷移が正常に行えない
- CSV出力ダイアログ表示またはCSV出力が正常に動作しない
- 不正状態操作時のエラー表示が不正である
- データ取得失敗時に未処理例外画面が表示される、または利用者向けエラー表示が出ない
- 他画面に共通CSS由来の重大な崩れが発生する
- 対象JS変更によりCSV出力や関連操作へ重大な不具合が発生する
- 短時間で安全に是正できない

## 7. 切り戻し手順

### 7.1 作業前
1. 発生事象、影響範囲、再現状況を確認する。  
2. 一時回避可否を判定する。  
3. 継続困難と判断した場合、切り戻し承認を得る。  
4. 切り戻し対象資材を確認する。  

### 7.2 切り戻し実施
1. 反映先環境へ接続する。  
2. 現在配置されている反映後資材を退避する。  
3. 反映前バックアップ資材を所定位置へ戻す。  
4. 必要に応じてアプリケーション再起動を行う。  
5. エラーログの有無を確認する。  

### 7.3 切り戻し後確認
1. `https://mock-task-app.example.com/Projects/＜ProjectId＞/Tasks` へアクセスする。  
2. タスク一覧画面が正常表示されることを確認する。  
3. 一覧表示、戻る導線、主要遷移、CSV出力の主要確認を行う。  
4. 取得失敗時表示および長文表示の重要観点を必要に応じて確認する。  
5. 他画面への影響が解消されていることを確認する。  
6. 関係者へ切り戻し完了を連絡する。
