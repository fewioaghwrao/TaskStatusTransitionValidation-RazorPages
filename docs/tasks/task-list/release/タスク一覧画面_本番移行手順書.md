## 2. 目的

本書は、タスク一覧画面のUI改修および操作性改善を本番環境へ安全に反映するための手順を整理したものである。  
あわせて、反映対象資材、反映後確認、障害発生時の対応、および切り戻し判断基準を明確にする。

※ 本資料は学習・模擬案件用に作成したものであり、対象URLおよび反映手順の一部は仮置きである。

---

## 3. 対象概要

### 3.1 対象画面
- タスク一覧画面（`/Projects/{projectId}/Tasks`）

### 3.2 対象機能
- 案件名を含む画面タイトル表示
- 状態遷移チェックに関する説明文表示
- 戻るリンクによる上位画面遷移
- タスク一覧表示
- 状態表示
- タスク詳細画面への遷移
- 新規タスク作成画面への遷移
- CSV出力ダイアログ表示
- CSV出力実行
- 不正状態操作時エラー表示
- データ取得失敗時エラー表示
- 0件 / 1件 / 複数件 / 長文データ表示
- 共通CSS / JS影響確認

### 3.3 改修概要
- 画面タイトルを案件名を含む表現へ見直し
- 状態遷移チェックに関する説明文の追加・調整
- 戻るリンク導線の追加・調整
- タスク一覧表示部のUI調整
- 状態表示の文言・見た目調整
- CSV出力導線およびダイアログ表示確認
- 通信例外時の利用者向けエラーメッセージ表示対応
- 長文表示時のレイアウト崩れ対策
- 共通CSSおよび対象JSの影響確認を含む画面調整

## 8. 本番反映手順

### 8.1 作業前
1. 反映対象資材を最終確認する。  
2. 配置先フォルダおよび上書き対象ファイルに誤りがないことを確認する。  
3. 現行版の対象資材をバックアップ取得する。  
4. 反映対象外資材が含まれていないことを確認する。  
5. 必要に応じて関係者へ作業開始を連絡する。  

### 8.2 資材反映
1. 反映先環境へ接続する。  
2. 現行の対象ファイルをバックアップ用フォルダへ退避する。  
3. 修正版資材を所定フォルダへ配置する。  
4. 必要に応じてアプリケーション再起動を行う。  
5. アプリ起動後、エラーログの有無を確認する。  

### 8.3 作業後
1. タスク一覧画面URLへアクセスする。  
2. 反映後確認チェックリストに沿って確認を行う。  
3. 問題がなければ反映完了とする。  
4. 問題がある場合は、内容を記録し、必要に応じて切り戻しを判断する。  

---

## 9. 反映後確認

反映後確認の詳細は、別紙「タスク一覧画面_反映後確認チェックリスト」を参照する。

主な確認観点は以下のとおり。

- 案件名を含むタイトル、説明文、戻るリンクが正しく表示されること
- 0件 / 1件 / 複数件 / 長文データでも一覧表示が崩れないこと
- 状態表示が正しい文言・見た目で表示されること
- タスク詳細画面、新規タスク作成画面への遷移が正しいこと
- CSV出力ダイアログ表示およびCSV出力が正常であること
- 不正状態操作時にエラー表示されること
- データ取得失敗時に利用者向けエラーメッセージが表示されること
- 共通CSS / JS影響により他画面や対象機能へ不整合が発生していないこと

---

## 10. 障害発生時の対応

1. 発生事象を確認する。  
2. 業務影響を判定する。  
3. 一時回避可否を確認する。  
4. 継続利用が困難な場合は切り戻しを検討する。  
5. 関係者へ事象・影響・対応方針を連絡する。  

---

## 11. 切り戻し判断条件

以下のいずれかに該当する場合、切り戻しを検討する。

- タスク一覧画面が表示されない
- 案件名を含むタイトル、説明文、戻るリンクが正常に表示されない
- タスク一覧が正常に取得・表示されない
- 0件 / 1件 / 複数件 / 長文表示で重大な崩れがある
- 状態表示が不正である
- タスク詳細画面または新規タスク作成画面への遷移が正常に行えない
- CSV出力ダイアログ表示またはCSV出力が正常に動作しない
- 不正状態操作時のエラー表示が不正である
- データ取得失敗時に未処理例外画面が表示される、または利用者向けエラー表示が出ない
- 共通CSS / JS影響により他画面または対象機能へ重大な不具合が発生している
- 短時間で安全に復旧できない

---

## 12. 注意事項

- 本番環境上で直接コード修正を行わないこと
- 旧版バックアップを取得せず反映しないこと
- docs配下の資料を反映対象へ含めないこと
- タスク一覧画面だけでなく、案件一覧画面・ログイン画面等への共通CSS影響も確認すること
- `projects-tasks.js` を反映対象とする場合は、CSV出力や関連操作への影響を重点確認すること
- 長文表示および取得失敗時表示は今回の重要確認ポイントとして必ず確認すること
- 異常発生時は独断で継続せず、必ず関係者へ共有すること
