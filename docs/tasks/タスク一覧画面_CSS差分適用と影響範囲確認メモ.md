# タスク一覧画面_CSS差分適用と影響範囲確認メモ

## 1. 目的
本メモは、タスク一覧画面のUI改修にあたり、CSS差分の適用方針と影響範囲確認の内容を整理することを目的とする。  
単なる見た目調整ではなく、既存画面を前提として、安全に差分を反映し、関連画面や共通スタイルへの不要な影響を抑えながら改修を進めるための確認記録として位置づける。

---

## 2. 対象画面
- 画面名: タスク一覧画面
- 対象URL例: `/Projects/{projectId}/Tasks`
- 対象機能:
  - 戻るリンク表示
  - 画面用途バッジ表示
  - ログインユーザー情報表示
  - 案件名表示
  - 状態遷移説明表示
  - タスク作成導線
  - 案件削除（アーカイブ）導線
  - 再読込導線
  - CSV出力導線
  - ローディング表示
  - 成功・エラーメッセージ表示
  - サマリー表示
  - キーワード・状態・優先度・期限・表示件数による絞り込み
  - 一覧表示
  - 状態変更フォーム
  - ページング
  - モバイル向けカード表示

---

## 3. 修正方針
今回の改修は、既存の業務意図、文言、主要な利用目的を原則維持しつつ、既存の一覧画面構成を大きく崩さずに、視認性・操作性・一覧性を改善する方針で実施した。

方針は以下のとおり。

- 既存の業務意図や文言は原則維持する
- 既存の一覧画面構成を大きく崩さない
- 画面上部で対象案件・利用者・用途を把握しやすくする
- 一覧表示前に状態把握ができるようサマリーを追加する
- 検索条件・件数表示・ページャを整理して操作しやすくする
- 状態や優先度は視認しやすいバッジで表示する
- 小画面ではカード表示へ切り替え、可読性を確保する
- 必要に応じて画面表示を支える軽微な処理を追加する
- 共通スタイルへの影響は極力抑え、画面専用CSSで吸収可能な部分は専用化する
- 追加する処理は、画面改善を成立させるための補助的な範囲に留める

---

## 4. CSS差分適用の考え方

### 4-1. 基本方針
CSS差分の適用にあたっては、タスク一覧画面としての一覧性・操作性・状態把握のしやすさを上げつつ、既存画面を安全に改修することを優先した。  
特に、ヘッダー、操作ボタン群、サマリー、検索フォーム、一覧表示、バッジ、ページャ、モバイル表示、CSV出力ダイアログを主対象とし、視認性と操作導線の向上を図った。

### 4-2. 主な修正対象
- 戻るリンク、画面識別バッジ、ユーザー表示、案件名を含む画面タイトル、説明文の整理
- タスク作成、案件削除、CSV出力、再読込の操作導線整理
- ローディング表示、成功メッセージ、エラーメッセージの追加・調整
- 期限・状態別件数を含むサマリー表示追加
- キーワード、状態、優先度、期限、表示件数による絞り込みフォーム整理
- テーブル形式の一覧表示調整
- 状態・優先度のバッジ表示調整
- モバイル時のカード表示追加
- ページャ、表示範囲表示、注記の整理
- CSV出力ダイアログの追加・調整

### 4-3. 共通CSSと画面専用CSSの切り分け
今回の改修では、タスク一覧画面固有の見た目改善は画面専用CSSで吸収し、共通スタイルの変更が必要な場合は他画面影響を確認する方針とした。

主な考え方:
- タスク一覧画面固有のレイアウト、サマリー、一覧、カード表示、CSVダイアログ等は `tasks.css` 側で対応する
- 共通ボタン、共通テーブル、共通バッジ、メッセージ、モーダル、レスポンシブ制御に関わる見た目は `site.css` 影響を意識して扱う
- JS で制御する CSV 出力、ローダー、ダイアログ関連は `projects-tasks.js` との整合を確認する
- 共通クラス変更が他画面に波及しないかを前提に確認する

## 5. 影響範囲として意識した対象

### 5-1. 直接影響対象
- タスク一覧画面のヘッダー表示
- 操作ボタン群表示
- ローディング・成功・エラー表示
- サマリー件数表示
- 検索・絞り込みフォーム
- 一覧表示
- 状態変更フォーム
- ページャ表示
- モバイル時のカード表示
- CSV出力ダイアログ表示

### 5-2. 間接影響対象
共通スタイル・共通JSが関係する可能性を考慮し、以下を間接影響対象として意識した。

- 案件一覧画面
- ログイン画面
- タスク詳細画面
- 新規タスク作成画面
- 共通ボタンスタイル
- 共通テーブルスタイル
- 共通バッジ、メッセージ、モーダルスタイル
- 共通レスポンシブ制御
- CSV出力関連操作
- 小画面表示

### 5-3. 特に確認が必要と考えた点
- 画面専用スタイルとして分離できる箇所が適切に分離されているか
- 共通クラス変更が他画面に意図せず影響していないか
- 絞り込み条件適用時と未適用時の両方で表示崩れがないか
- 件数が少ない場合、多い場合の両方で一覧表示が崩れないか
- 小画面でカード表示へ正しく切り替わるか
- 状態変更可否表示と実処理が整合しているか
- CSV出力ダイアログ表示および CSV 出力が正常であるか
- 取得失敗時表示および長文表示で重大な崩れがないか

---

## 6. 確認観点

### 6-1. タスク一覧画面単体確認
- 戻るリンク、バッジ、タイトル、説明文が正しく表示されること
- ログインユーザー情報、案件名、用途説明が把握しやすいこと
- 操作ボタン群が整理され、主操作と補助操作の区別がつくこと
- ローディング表示、成功メッセージ、エラーメッセージが適切に表示されること
- 期限・状態別件数サマリーが想定どおり表示されること
- 検索・絞り込み・表示件数変更・ページャが正しく動作すること
- 状態表示、優先度表示が視認しやすいこと
- Done タスクが更新不可となるなど、状態変更可否が画面表示と整合すること
- モバイル時にカード表示へ切り替わること
- CSV出力ダイアログが正しく表示され、CSV が想定どおり出力されること
- 0件 / 1件 / 複数件 / 長文表示で重大な崩れがないこと
- 取得失敗時に未処理例外画面ではなく利用者向けエラーメッセージが表示されること

### 6-2. 横断確認
- 案件一覧画面の検索欄、ボタン、一覧表示、ページャに不自然な崩れがないこと
- ログイン画面の入力欄、ボタン、補助文言に不自然な波及がないこと
- タスク詳細画面、新規タスク作成画面への遷移および表示に重大な不整合がないこと
- 共通ボタン、共通バッジ、メッセージ、モーダル、レスポンシブ制御に重大な崩れがないこと
- CSV出力や関連操作に JS 起因の重大な不具合がないこと

---

## 7. 確認結果
タスク一覧画面に対して、総合テストケースに基づく手動確認を実施し、実施件数15件、OK 15件、NG 0件を確認した。  
基本導線、一覧表示、CSV出力、0件 / 1件表示、取得失敗時表示、長文表示、他画面への共通CSS影響について確認を行い、重大な問題は確認されなかった。  
取得失敗時表示は、初回確認時の問題に対して通信例外ハンドリング修正後の再確認を行い、利用者向けエラーメッセージが画面上に表示されることを確認した。  
また、長文表示についても CSS 修正後の再確認により表示崩れが解消されていることを確認した。

このため、タスク一覧画面を中心とした CSS / JS 差分は、既存画面改修として許容範囲内で安全に反映できると判断した。

## 8. 今回の改修で示せること
本対応により、以下の観点を示せる。

- 既存画面を前提とした最小差分でのUI改修
- 共通CSSと画面専用CSSの切り分け意識
- 一覧画面の視認性・操作性・状態把握改善の進め方
- 表示改善だけでなく、検索・ページング・状態変更・CSV出力・異常系表示まで含めた確認整理
- 他画面影響と共通CSS / JS 影響を意識した横断確認の進め方
- UI修正と補助処理整理を分けて扱う考え方
