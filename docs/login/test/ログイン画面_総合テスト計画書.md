# ログイン画面_総合テスト計画書

## 1. 目的

ログイン画面の画面変更、およびログイン機能に関して、  
利用者が業務上問題なく利用できることを総合テストで確認する。

本テストでは、以下を主に確認対象とする。

- 画面レイアウトが設計意図どおりに表示されること
- 入力項目、ボタン、メッセージ表示が正しく動作すること
- 認証成功時／失敗時の画面遷移・表示が正しいこと
- 画面崩れや操作不能など、UI変更による不具合が発生していないこと

## 2. 対象

### 2.1 対象画面

- ログイン画面

### 2.2 対象機能

- ユーザーID入力
- パスワード入力
- ログイン実行
- 入力エラー表示
- 認証失敗時メッセージ表示
- 認証成功後の遷移

## 3. テスト範囲

### 3.1 実施対象

- 画面表示確認
- 入力・操作確認
- メッセージ表示確認
- 画面遷移確認
- PC画面幅における表示確認
- 狭幅表示時の簡易レイアウト確認
- 共通CSS影響の有無確認（必要に応じて関連画面を目視確認）

### 3.2 実施対象外

- 認証基盤自体の詳細内部仕様検証
- 性能試験
- 脆弱性診断
- ブラウザ全網羅試験
- 本番環境相当の大規模負荷試験

## 4. 前提条件

- 対象アプリが起動可能であること
- テスト用ユーザーが払い出されていること
- 正常系確認用の有効なユーザーID／パスワードを準備済みであること
- 異常系確認用として、無効なユーザーID／パスワードの組み合わせを準備済みであること

## 5. テスト観点

- 初期表示
- 項目表示
- 項目入力
- 必須入力
- 認証成功
- 認証失敗
- エラーメッセージ
- ボタン操作
- Enterキー操作
- Tabキーによるフォーカス移動
- 連続操作時の挙動
- レイアウト崩れ
- 文言整合性
- 共通CSS影響確認
- 戻る／再表示時の挙動（必要に応じて）

## 6. 確認環境

| 項目 | 内容 |
|---|---|
| OS | Windows 11 |
| ブラウザ | Chrome（シークレットウィンドウ） |
| 画面幅 | PC標準幅、必要に応じて狭幅確認 |
| 確認方法 | 手動目視確認 |
| 対象URL | 確認対象環境のURL |

## 7. 実施手順

1. ログイン画面を表示する
2. 初期表示を確認する
3. 正常系・異常系の入力パターンを実施する
4. 必須入力チェック、認証失敗時メッセージを確認する
5. 認証成功後の遷移先画面を確認する
6. Tab移動、Enterキー操作、連続操作時の挙動を確認する
7. レイアウト崩れや文言差異がないか確認する
8. 必要に応じて関連画面に共通CSS影響がないか確認する
9. 結果をテストケースに記録する

## 8. 判定基準

- 期待結果どおりであれば「OK」
- 軽微な差異がある場合は内容を備考に記載する
- 期待結果と異なる場合は「NG」とし、事象を記録する
- 誤記等の軽微な記録ミスがある場合は、確認実態に問題がなければ備考で補足のうえ「OK」とする

## 9. 成果物

- 確認観点一覧.tsv
- 総合テストケース.tsv
- 実施結果.md

## 10. 留意事項

- ログイン失敗メッセージは、セキュリティ観点上、過度に詳細な文言でないことも確認する
- UI変更が共通スタイルに影響する場合は、他画面への影響も別途確認対象とする
- 長文メッセージやエラー表示時にレイアウト崩れがないか注意する
- 今回の確認は Chrome を用いた手動確認を基本とし、他ブラウザ確認は必要時に別途実施する
