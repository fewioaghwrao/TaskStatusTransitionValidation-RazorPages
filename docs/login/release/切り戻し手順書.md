## 4. 切り戻し判断条件

以下のいずれかに該当する場合、切り戻しを検討する。

- ログイン画面が正常に表示されない
- メールアドレスまたはパスワード入力が正常に行えない
- ログインボタン押下後に正常動作しない
- 正常ログイン後に `/Projects/Index` へ遷移しない
- 認証失敗時や例外時のメッセージ表示が不正
- パスワード表示切替が動作しない
- バリデーション未通過時に送信中表示へ切り替わるなど、重大な操作不整合がある
- 二重送信防止が機能しない
- 他画面に共通CSS由来の重大な崩れが発生する
- 短時間で安全に是正できない

## 7. 切り戻し手順

### 7.1 作業前
1. 発生事象、影響範囲、再現状況を確認する。  
2. 一時回避可否を判定する。  
3. 継続困難と判断した場合、切り戻し承認を得る。  
4. 切り戻し対象資材を確認する。  

### 7.2 切り戻し実施
1. 反映先環境へ接続する。  
2. 現在配置されている反映後資材を退避する。  
3. 反映前バックアップ資材を所定位置へ戻す。  
4. 必要に応じてアプリケーション再起動を行う。  
5. エラーログの有無を確認する。  

### 7.3 切り戻し後確認
1. `https://mock-task-app.example.com/Login` へアクセスする。  
2. ログイン画面が正常表示されることを確認する。  
3. 正常ログイン、異常ログインの主要確認を行う。  
4. 他画面への影響が解消されていることを確認する。  
5. 関係者へ切り戻し完了を連絡する。
