## 2. 目的

本書は、ログイン画面のUI改修および軽微な挙動補正を本番環境へ安全に反映するための手順を整理したものである。  
あわせて、反映対象資材、反映後確認、障害発生時の対応、および切り戻し判断の基準を明確にする。

※ 本資料は学習・模擬案件用に作成したものであり、対象URLおよび反映手順の一部は仮置きである。

---

## 3. 対象概要

### 3.1 対象画面
- ログイン画面

### 3.2 対象機能
- メールアドレス入力
- パスワード入力
- ログイン実行
- バリデーションエラー表示
- 認証失敗メッセージ表示
- ログイン成功後の画面遷移
- パスワード表示切替
- 送信中表示制御
- 二重送信防止

### 3.3 改修概要
- 画面上部のバッジ、タイトル、説明文の追加・調整
- 入力欄、ラベル、エラーメッセージの視認性改善
- パスワード表示切替ボタン追加
- ログインボタンの見た目強調
- バリデーション未通過時に送信中表示へ切り替わらないよう補正
- ブラウザ戻る操作後に送信ボタン状態を通常へ戻す補正
- 共通CSS影響の確認対象を含む画面調整

## 8. 本番反映手順

### 8.1 作業前
1. 反映対象資材を最終確認する。  
2. 現行版の対象資材をバックアップ取得する。  
3. 反映対象外資材が含まれていないことを確認する。  
4. 必要に応じて関係者へ作業開始を連絡する。  

### 8.2 資材反映
1. 反映先環境へ接続する。  
2. 現行の対象ファイルをバックアップ用フォルダへ退避する。  
3. 修正版資材を所定フォルダへ配置する。  
4. 必要に応じてアプリケーション再起動を行う。  
5. アプリ起動後、エラーログの有無を確認する。  

### 8.3 作業後
1. ログイン画面URLへアクセスする。  
2. 反映後確認チェックリストに沿って確認を行う。  
3. 問題がなければ反映完了とする。  
4. 問題がある場合は、内容を記録し、必要に応じて切り戻しを判断する。

## 10. 障害発生時の対応
1. 発生事象を確認する。  
2. 業務影響を判定する。  
3. 一時回避可否を確認する。  
4. 継続利用が困難な場合は切り戻しを検討する。  
5. 関係者へ事象・影響・対応方針を連絡する。  

---

## 11. 切り戻し判断条件
以下のいずれかに該当する場合、切り戻しを検討する。

- ログイン画面が表示されない
- メールアドレス／パスワード入力が正常に行えない
- ログイン処理が正常に実行できない
- 正常ログイン後に `/Projects/Index` へ遷移しない
- 認証失敗時や例外時のメッセージ表示が不正
- パスワード表示切替が動作しない
- 二重送信防止が機能しない
- 共通CSS影響により他画面へ重大な崩れが発生している
- 短時間で安全に復旧できない

---

## 12. 注意事項
- 本番環境上で直接コード修正を行わないこと
- 旧版バックアップを取得せず反映しないこと
- docs配下の資料を反映対象へ含めないこと
- ログイン画面だけでなく、関連画面への共通CSS影響も確認すること
- 異常発生時は独断で継続せず、必ず関係者へ共有すること
