# 案件一覧画面_修正概要

## 1. 概要

本資料は、既存の Razor Pages 制作物である  
[TaskStatusTransitionValidation_Razor](https://github.com/fewioaghwrao/TaskStatusTransitionValidation_Razor)  
に含まれる **案件一覧画面** を対象として実施した画面修正の概要を整理したものです。

今回の対応では、文言や業務仕様は変更せず、既存UIの見直しの一環として CSS を中心に調整を行い、**視認性** と **操作性** の向上を図りました。

---

## 2. 背景

対象画面は、既存リポジトリ `TaskStatusTransitionValidation_Razor` に含まれる案件一覧画面です。

既存画面は機能としては成立していたものの、以下のような改善余地がありました。

- 画面タイトルやサブタイトルの視認性がやや弱い
- 検索条件エリアの入力欄やボタンの役割差が見えにくい
- 一覧行ごとの区切りや補足情報の見え方がやや弱い
- ページャや注釈文の余白・配色に改善余地がある

そのため、既存レイアウトや構成を踏まえたうえで、画面修正の一環として案件一覧画面の CSS 修正を実施しました。

---

## 3. 対象画面

- 画面名: 案件一覧画面
- 主な対象領域:
  - ヘッダー部
  - 検索条件エリア
  - 案件一覧表示部
  - ページャ
  - 注釈文

---

## 4. 修正目的

本修正の目的は、案件一覧画面における **視認性** および **操作性** の向上です。

特に、以下の観点を重視しました。

- 画面タイトルや説明文を見やすくする
- 検索操作と補助操作の区別を明確にする
- 一覧表示の区切りや情報の見やすさを高める
- 主要操作ボタンを強調し、画面全体のメリハリをつける
- 既存画面の構成を大きく崩さずに見た目を改善する

---

## 5. 修正方針

本修正では、**文言や業務仕様は変更せず**、主に以下の観点でUI調整を実施しました。

- 配色の見直し
- 文字サイズの調整
- 余白の再設計
- ボタンの強調表現の整理
- 既存共通スタイルへの影響を考慮した画面単位のCSS分離

また、共通クラスに影響する箇所は必要性を見極めながら調整し、画面固有の見直しが必要な部分については専用CSSを追加する方針としました。

---

## 6. 主な修正内容

### 6.1 ヘッダー部

- 画面タイトルの見やすさを向上
- サブタイトルの文字色と文字サイズを見直し、背景とのコントラストを調整
- 案内バッジの余白と文字色を調整し、画面種別の識別性を向上
- ユーザー表示部についても背景色や文字サイズを見直し、情報の見やすさを改善

### 6.2 検索条件エリア

- 検索入力欄の背景色、角丸、境界線を見直し、入力領域を明確化
- 検索ボタンは主操作として識別しやすいよう、背景色・枠線色・文字色を調整
- クリアボタンは補助操作として認識しやすいよう、検索ボタンとの差別化を実施
- 件数切替のセレクトボックスについても背景色や文字サイズを調整
- 件数表示の文字色や余白を見直し、検索結果情報を読みやすくした

### 6.3 案件一覧表示部

- 一覧行の背景色と内側余白を調整し、案件ごとの区切りを明確化
- 案件リンクの見た目を見直し、選択可能な項目であることを分かりやすくした
- 補足情報の文字色を調整し、背景に埋もれにくいよう改善
- 状態表示の内側余白を見直し、視認性を向上

### 6.4 ページャ・注釈文

- ページャ上部の余白やボタン間隔を見直し、操作エリアのまとまりを改善
- 注釈文の文字色を調整し、背景上でも読みやすくした

- ボタン文言
- 検索プレースホルダ
- 件数表示文言
- 注釈文

### 7.2 業務仕様変更
業務仕様の変更は行っていません。

今回の対応は、主に見た目および操作性の改善を目的とした **UI調整** です。

## 8. 修正の位置づけ

今回の対応は、既存リポジトリ `TaskStatusTransitionValidation_Razor` の画面改善の一環として実施したものです。

単なる新規画面作成ではなく、既存画面を前提として以下を意識しました。

- 既存レイアウト・構成の把握
- 修正差分の整理
- 共通スタイルへの影響確認
- 画面単位での改善ポイントの抽出
- Before / After による見た目比較資料の整理

このため、本対応は **既存画面改修** と **画面設計書修正観点の整理** の両面を持つ取り組みとして位置づけています。
