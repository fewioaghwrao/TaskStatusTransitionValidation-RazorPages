# 案件一覧画面_CSS差分適用と影響範囲確認メモ

## 1. 目的
本メモは、案件一覧画面のUI改修にあたり、CSS差分の適用方針と影響範囲確認の内容を整理することを目的とする。  
単なる見た目調整ではなく、既存画面を前提として、安全に差分を反映し、関連画面や共通スタイルへの不要な影響を抑えながら改修を進めるための確認記録として位置づける。

---

## 2. 対象画面
- 画面名: 案件一覧画面
- 対象URL: `/Projects/Index`
- 対象機能:
  - 案件一覧表示
  - 件数表示
  - 案件検索
  - ページサイズ変更
  - ページング
  - 案件詳細（タスク一覧）への遷移
  - 新規作成画面への遷移
  - 再読み込み
  - 空データ時表示
  - エラー時表示
  - ローディング表示
  - ログイン状態チェック
  - 権限に応じた表示制御
  - 不正パラメータ時の表示制御

---

## 3. 修正方針
今回の改修は、既存の業務仕様、検索仕様、ページング仕様、画面遷移仕様を維持しつつ、既存レイアウトを大きく崩さずに、見やすさ・使いやすさを改善する方針で実施した。

方針は以下のとおり。

- 文言や業務意図は原則維持する
- 業務仕様の大幅変更は行わない
- 既存レイアウトを大きく崩さない
- 主に CSS を中心とした画面改修とし、必要最小限の構造整理に留める
- 変更差分は画面単位で整理し、画面専用スタイルで吸収可能なものは専用CSSで対応する
- 共通スタイルを変更する場合は、他画面への影響確認を前提とする
- 実装内容は総合テスト資料および本番移行資料と整合するよう整理する

---

## 4. CSS差分適用の考え方

### 4-1. 基本方針
CSS差分の適用にあたっては、一覧画面としての可読性と操作性を上げつつ、既存画面を安全に改修することを優先した。  
特に、ヘッダー、検索条件エリア、一覧表示部、ページャ、注釈文を主対象とし、視認性と操作性の向上を図った。

### 4-2. 主な修正対象
- 画面上部のバッジ、タイトル、説明文の追加・調整
- 検索条件エリアの入力欄、検索ボタン、クリアボタンの見た目調整
- 件数表示、ページサイズ選択、ページャ表示の追加・調整
- 案件一覧カードのレイアウト改善
- 案件リンク押下によるタスク一覧画面遷移の導線整理
- Leader 権限時のみ新規作成ボタンを表示する制御追加
- 再読み込み導線の追加
- 空データ時メッセージ、エラーメッセージ、ローディング表示の追加・調整
- ユーザー情報表示調整（該当する場合）

### 4-3. 共通CSSと画面専用CSSの切り分け
今回の改修では、案件一覧画面固有の見た目改善は画面専用CSSで吸収し、共通スタイルの変更が必要な場合は他画面影響を確認する方針とした。

主な考え方:
- 一覧画面のレイアウトや検索エリア、一覧カードなどは `projects.css` 側で対応する
- 共通入力スタイル、共通ボタンスタイル、共通テキスト色やリンクスタイルに関わる修正は `site.css` 側の影響を意識して扱う
- `_UserBox.cshtml` を含む共通表示要素を変更する場合は、他画面への波及を前提に確認する

---

## 5. 影響範囲として意識した対象

### 5-1. 直接影響対象
- 案件一覧画面のヘッダー表示
- 検索条件入力欄
- 検索ボタン、クリアボタン
- 件数切替表示
- 案件一覧行の見え方
- 案件リンク表示
- ページャ表示
- 注釈文表示
- ローディング表示
- エラー表示
- 空データ時表示

### 5-2. 間接影響対象
共通スタイルが関係する可能性を考慮し、以下を間接影響対象として意識した。

- ログイン画面
- タスク一覧画面
- 案件作成画面
- 共通入力スタイル
- 共通ボタンスタイル
- 共通テキスト色／リンクスタイル
- `_UserBox.cshtml` を利用する画面
- 横幅縮小時の表示

### 5-3. 特に確認が必要と考えた点
- 共通入力スタイル変更が他画面の検索欄や入力欄に波及しないか
- 共通ボタンスタイル変更が主操作・補助操作ボタンの見え方を崩さないか
- 共通テキスト色やリンクスタイル変更が他画面の一覧表示や補助文言に波及しないか
- 検索条件エリア、一覧表示、ページャで崩れがないか
- 横幅縮小時やデータ件数変動時にも不自然な表示にならないか

---

## 6. 確認観点

### 6-1. 案件一覧画面単体確認
- 初期表示時にタイトル、件数、一覧、検索欄、各種ボタンが正常表示されること
- バッジ、メインタイトル、サブタイトル、ユーザー情報が正しく表示されること
- 読み込み中パネルが表示され、読み込み完了後に非表示となること
- データ0件時に空データ時メッセージが表示されること
- ユーザー情報取得失敗時にエラーメッセージが表示され、画面継続表示となること
- 検索条件入力・検索結果表示が正しいこと
- クリア動作が正しいこと
- ページサイズ変更時に想定どおり先頭ページへ戻ること
- ページング動作が正しいこと
- 案件リンク押下時にタスク一覧画面へ遷移できること
- Leaderユーザーでは「＋ 新規作成」ボタンが表示されること
- Workerユーザーでは新規作成ボタンが表示されないこと
- 再読み込み動作が正しいこと
- 未ログイン時および API 例外時にログイン画面遷移が正しいこと

### 6-2. 横断確認
- ログイン画面の入力欄、ボタン、補助文言に不自然な崩れがないこと
- タスク一覧画面の検索欄、セレクトボックス、ボタン、一覧表示、ページャに崩れがないこと
- 案件作成画面への遷移および表示に重大な不整合がないこと
- 共通ユーザー表示部品に不自然な変化がないこと
- 横幅縮小時にも主要要素が操作不能にならないこと
- 共通CSS影響により他画面へ重大な崩れが発生していないこと

---

## 7. 確認結果
案件一覧画面に対して、総合テストケースに基づく確認を実施し、画面表示、権限制御、検索、画面遷移、ページング、画面制御、認証、データ制御、入力制御の計45件について全件OKを確認した。  
UI改修に伴う画面崩れ、操作不能、誤遷移などの重大不具合は確認されなかった。

また、共通CSS由来の影響確認対象として、ログイン画面、タスク一覧画面等の関連画面を意識し、他画面に重大な崩れが発生していないことを確認対象に含めた。  
このため、案件一覧画面を中心としたCSS差分は、既存画面改修として許容範囲内で安全に反映できると判断した。

---

## 8. 今回の改修で示せること
本対応により、以下の観点を示せる。

- 既存画面を前提とした最小差分でのUI改修
- 共通CSSと画面専用CSSの切り分け意識
- 一覧画面の視認性・操作性改善の進め方
- 表示改善だけでなく、検索・ページング・遷移・権限制御まで含めた確認整理
- 他画面影響を意識した横断確認の進め方
- UI修正と軽微な画面整理を分けて扱う考え方
