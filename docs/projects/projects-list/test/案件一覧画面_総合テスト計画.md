# 案件一覧画面_総合テスト計画

## 1. 目的
案件一覧画面の画面表示、および一覧表示・検索・ページング・画面遷移に関して、
利用者が業務上問題なく利用できることを総合テストで確認する。

本テストでは、以下を主に確認対象とする。

- 画面レイアウトが設計意図どおりに表示されること
- 案件一覧、件数表示、検索、ページサイズ変更、ページングが正しく動作すること
- ログイン状態に応じた表示・遷移が正しいこと
- 権限に応じた新規作成ボタン表示が正しいこと
- エラーメッセージ、空データ時表示、ローディング表示が正しく動作すること
- 画面崩れや操作不能など、UI変更や一覧機能追加による不具合が発生していないこと

## 2. 対象

### 2.1 対象画面
- 案件一覧画面（/Projects/Index）

### 2.2 対象機能
- 案件一覧表示
- ログイン状態チェック
- ユーザー情報表示
- 案件件数表示
- 案件検索
- ページサイズ変更
- ページング
- 案件詳細（タスク一覧）への遷移
- 新規作成画面への遷移
- 再読み込み
- 空データ時表示
- エラー時表示
- ローディング表示

## 3. テスト方針
本テストは、単体テスト完了後または画面修正後の総合テストとして実施する。

主に以下の観点で確認する。

- 画面表示確認
- 入力／操作確認
- 画面遷移確認
- 条件分岐確認
- 異常系確認
- 表示件数／ページング確認
- 権限制御確認

## 4. 前提条件
- アプリケーションが起動済みであること
- APIが疎通可能であること
- 認証用Cookie（auth_token）の有無を確認できること
- テスト用ユーザーとして以下を準備すること
  - Leader権限ユーザー
  - Worker権限ユーザー
- テスト用案件データを必要件数準備すること
  - 0件
  - 1件
  - 複数件（5件超、10件超）
  - アーカイブ案件を含むデータ
  - 検索にヒットする案件／しない案件
- 必要に応じてAPI異常またはユーザー情報取得失敗を再現できること

## 5. 確認対象外
今回の総合テストでは、以下は主対象外とする。

- API内部の詳細ロジック単体確認
- DBレベルの性能試験
- セキュリティ診断
- ブラウザ間の詳細互換性試験
- 長時間連続稼働試験

## 6. 主な確認観点
- 初期表示
- タイトル、サブタイトル、バッジ表示
- ユーザー情報表示
- エラーメッセージ表示
- 案件件数表示
- 案件一覧表示
- 案件0件時表示
- 検索条件入力・検索結果表示
- クリア動作
- ページサイズ変更
- ページング動作
- 案件リンク押下時遷移
- 新規作成ボタンの表示制御と遷移
- 再読み込み動作
- ローディング表示
- 未ログイン時のログイン画面遷移
- API異常時のログイン画面遷移
- ユーザー情報取得失敗時のエラー表示

## 7. 実施成果物
- 案件一覧画面_総合テスト計画.md
- 確認観点一覧.tsv
- 総合テストケース.tsv
- 必要に応じて実施結果記録
- 不具合管理票

## 8. 判定基準
- 期待結果を満たすこと
- 業務利用上問題となる表示崩れ、操作不能、誤遷移がないこと
- 異常時に利用者が継続不能となる重大不具合がないこと

## 9. 留意事項
- 本画面では一覧対象は実質的に非アーカイブ案件の表示が中心であるため、
  アーカイブ案件の扱いは表示仕様と実装差異がないか注意して確認する
- 検索はプロジェクト名・説明文を対象としているため、両方でヒット確認を行う
- ページサイズ変更時は hidden の Page=1 が送信されるため、先頭ページへ戻ることを確認する
- ローディング表示は画面ロード時およびリンク・フォーム送信時に表示されることを確認する
