## 2. 目的

本書は、案件一覧画面のUI改修および一覧機能改善を本番環境へ安全に反映するための手順を整理したものである。  
あわせて、反映対象資材、反映後確認、障害発生時の対応、および切り戻し判断基準を明確にする。

※ 本資料は学習・模擬案件用に作成したものであり、対象URLおよび反映手順の一部は仮置きである。

---

## 3. 対象概要

### 3.1 対象画面
- 案件一覧画面（`/Projects/Index`）

### 3.2 対象機能
- 案件一覧表示
- ユーザー情報表示
- 件数表示
- 案件検索
- ページサイズ変更
- ページング
- タスク一覧画面への遷移

- 再読み込み

- エラー時表示
- ローディング表示
- ログイン状態チェック

- 不正パラメータ時の表示制御

### 3.3 改修概要
- 画面上部のバッジ、タイトル、説明文の追加・調整
- 案件一覧カードのレイアウト改善
- 件数表示、検索フォーム、ページサイズ選択、ページャ表示の追加・調整
- Leader 権限時のみ新規作成ボタンを表示する制御追加
- 案件リンク押下によるタスク一覧画面遷移の導線整理
- 再読み込み導線の追加
- 空データ時メッセージ、エラーメッセージ、ローディング表示の追加・調整

- アーカイブ案件除外表示および不正 `page` / `pageSize` パラメータ時の補正確認
- 共通CSS影響の確認対象を含む画面調整

## 8. 本番反映手順

### 8.1 作業前
1. 反映対象資材を最終確認する。  
2. 配置先フォルダおよび上書き対象ファイルに誤りがないことを確認する。  
3. 現行版の対象資材をバックアップ取得する。  
4. 反映対象外資材が含まれていないことを確認する。  
5. 必要に応じて関係者へ作業開始を連絡する。  

### 8.2 資材反映
1. 反映先環境へ接続する。  
2. 現行の対象ファイルをバックアップ用フォルダへ退避する。  
3. 修正版資材を所定フォルダへ配置する。  
4. 必要に応じてアプリケーション再起動を行う。  
5. アプリ起動後、エラーログの有無を確認する。  

### 8.3 作業後
1. 案件一覧画面URLへアクセスする。  
2. 反映後確認チェックリストに沿って確認を行う。  
3. 問題がなければ反映完了とする。  
4. 問題がある場合は、内容を記録し、必要に応じて切り戻しを判断する。  

---

## 9. 反映後確認

反映後確認の詳細は、別紙「案件一覧画面_反映後確認チェックリスト」を参照する。

主な確認観点は以下のとおり。

- 初期表示が正常であること
- 案件一覧、件数表示、検索、ページサイズ変更、ページングが正しく動作すること
- Leader / Worker に応じた新規作成ボタン表示が正しいこと
- 案件リンク、案件作成、再読み込みの遷移が正しいこと
- 空データ時表示、エラー時表示、ローディング表示が正しいこと
- 未ログイン時および API 例外時の遷移が正しいこと
- アーカイブ案件が一覧対象外となること
- 不正 `pageSize` / `page` 指定時も画面継続利用が可能であること
- 共通CSS影響により他画面へ崩れが発生していないこと

---

## 10. 障害発生時の対応

1. 発生事象を確認する。  
2. 業務影響を判定する。  
3. 一時回避可否を確認する。  
4. 継続利用が困難な場合は切り戻しを検討する。  
5. 関係者へ事象・影響・対応方針を連絡する。  

---

## 11. 切り戻し判断条件

以下のいずれかに該当する場合、切り戻しを検討する。

- 案件一覧画面が表示されない
- 案件一覧が正常に取得・表示されない
- 件数表示が不正である
- 検索、ページサイズ変更、ページングが正常に動作しない
- タスク一覧画面または案件作成画面への遷移が正常に行えない
- 権限に応じた表示制御が不正である
- 未ログイン時または API 例外時の遷移が不正である
- 空データ時表示、エラー表示、ローディング表示に重大な不整合がある
- アーカイブ案件の表示制御が不正である
- 不正パラメータ指定時に画面利用継続ができない
- 共通CSS影響により他画面へ重大な崩れが発生している
- 短時間で安全に復旧できない

---

## 12. 注意事項

- 本番環境上で直接コード修正を行わないこと
- 旧版バックアップを取得せず反映しないこと
- docs配下の資料を反映対象へ含めないこと
- 案件一覧画面だけでなく、関連画面への共通CSS影響も確認すること
- Leader / Worker の両権限で主要確認を行うこと
- 検索欄プレースホルダ文言と実装内容の差異に注意すること
- 異常発生時は独断で継続せず、必ず関係者へ共有すること
