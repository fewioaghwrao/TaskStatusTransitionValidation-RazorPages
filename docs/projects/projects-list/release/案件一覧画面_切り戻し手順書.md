## 2. 目的

本書は、案件一覧画面の本番反映後に重大な不具合が発生した場合に、速やかに反映前の状態へ戻すための手順を定めるものである。

※ 本資料は学習・模擬案件用に作成したものであり、環境情報の一部は仮置きである。

---

## 3. 切り戻し対象

原則として、今回反映した資材一式を切り戻し対象とする。  
影響範囲が明確であり、整合性が保てる場合に限り、対象資材のみ個別切り戻しを可とする。

### 対象資材
- `TaskStatusTransitionValidation.RazorMock/Pages/Projects/Index.cshtml`
- `TaskStatusTransitionValidation.RazorMock/Pages/Projects/Index.cshtml.cs`
- `TaskStatusTransitionValidation.RazorMock/Pages/Shared/_UserBox.cshtml`
- `TaskStatusTransitionValidation.RazorMock/wwwroot/css/site.css`
- `TaskStatusTransitionValidation.RazorMock/wwwroot/css/projects.css`

※ `_UserBox.cshtml`、`projects.css` は今回反映対象に含まれる場合のみ切り戻し対象とする。

---

## 4. 切り戻し判断条件

以下のいずれかに該当する場合、切り戻しを検討する。

- 案件一覧画面が正常に表示されない
- 案件一覧が正常に取得・表示されない
- 件数表示が不正である
- 検索、ページサイズ変更、ページングが正常に動作しない
- タスク一覧画面または案件作成画面への遷移が正常に行えない
- 権限に応じた表示制御が不正である
- 未ログイン時または API 例外時の遷移が不正である
- 空データ時表示、エラー表示、ローディング表示に重大な不整合がある
- アーカイブ案件の表示制御が不正である
- 不正パラメータ指定時に画面利用継続ができない
- 他画面に共通CSS由来の重大な崩れが発生する
- 短時間で安全に是正できない

## 7. 切り戻し手順

### 7.1 作業前
1. 発生事象、影響範囲、再現状況を確認する。  
2. 一時回避可否を判定する。  
3. 継続困難と判断した場合、切り戻し承認を得る。  
4. 切り戻し対象資材を確認する。  

### 7.2 切り戻し実施
1. 反映先環境へ接続する。  
2. 現在配置されている反映後資材を退避する。  
3. 反映前バックアップ資材を所定位置へ戻す。  
4. 必要に応じてアプリケーション再起動を行う。  
5. エラーログの有無を確認する。  

### 7.3 切り戻し後確認
1. `https://mock-task-app.example.com/Projects/Index` へアクセスする。  
2. 案件一覧画面が正常表示されることを確認する。  
3. 案件一覧、検索、ページング、主要遷移の主要確認を行う。  
4. 権限制御および認証系の主要確認を行う。  
5. 他画面への影響が解消されていることを確認する。  
6. 関係者へ切り戻し完了を連絡する。
